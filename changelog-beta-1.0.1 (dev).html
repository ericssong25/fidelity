NUEVO Lógica de Canje con min_level

Tab Canjear rediseñado

  • Layout de 2 columnas: izquierda (ícono + nombre + descripción truncada + badge puntos) | derecha (condiciones + botón)
  • Orden automático: canjeables → sin puntos → nivel insuficiente → agotados/expirados al final
  • Máx 2 badges por card

Diseño de cada reward card

  • Normal: ícono Gift, nombre bold, descripción line-clamp-2, badge de puntos
  • Exclusivo (Silver/Gold): ícono Star/Crown, ribbon "Exclusivo [Nivel]" con color del nivel
  • Nivel insuficiente: ícono Lock, card opacity-60, botón gris "Nivel Silver requerido"
  • Limitado: badge rojo "Quedan X", pulse si ≤ 3, overlay "Agotado" si stock = 0
  • Temporal: ícono reloj, "Últimas Xh" en rojo si < 3 días, overlay "Expirado" si venció

Estados del botón

  • Canjeable: "Canjear por X pts" filled #7546ED
  • Sin puntos: "Te faltan X pts" outline #B1A9E5
  • Nivel insuficiente: "Nivel [X] requerido" gris
  • Agotado / Expirado: botón gris con texto correspondiente

Modal de confirmación mejorado

  • Ícono Gift + nombre + descripción + "¿Canjear por X puntos?" + "Tendrás Y pts restantes"
  • handleRedeem verifica nivel antes de puntos
  • Solo descuenta current_points — nunca toca total_points_earned

Panel Negocio — min_level en rewards

  • Selector toggle horizontal tipo segmented control: [Bronze] [Silver] [Gold]
  • Cada opción con color del nivel al seleccionarse. Bronze por defecto = sin restricción (null en DB)
  • Badges visuales en las cards del panel: "Exclusivo Silver/Gold", "Quedan X", rango de fechas
